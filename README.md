## ⚠️ 중요: 데이터 저장 방식 변경 (v1.5)
- **하이브리드 아키텍처 (온/오프라인 지원):** **Firebase(클라우드)**와 **IndexedDB(로컬)**를 함께 사용합니다.
  - **온라인 시:** 모든 데이터는 Firebase와 실시간으로 동기화됩니다.
  - **비용 최적화:** Firebase 무료 용량 초과를 방지하기 위해, 실시간 동기화 대신 앱 시작 시와 '수동 동기화' 버튼 클릭 시에만 데이터를 가져오도록 변경되었습니다.
  - **오프라인 시:** 출석 기록 등 모든 변경사항은 기기 내부에 안전하게 저장되며, 상단에 '오프라인 모드'가 표시됩니다.
  - **자동 동기화:** 인터넷 연결이 복구되면, 오프라인 상태에서 저장된 기록들이 자동으로 Firebase 서버에 업로드되고 '동기화 완료' 알림이 표시됩니다.
- **데이터 안전성:** 이제 데이터는 클라우드와 로컬에 이중으로 저장되어 더욱 안전합니다. 하지만 정기적인 데이터 백업을 권장합니다.

- **제품명:** 몸펴기생활운동 연신내 출석부

- **회원 출석 모드:**

- **종이 명부 입력 (Bulk Input):** 
  - 수기로 작성된 명단을 텍스트로 붙여넣기하여 일괄 출석 처리.
  - 형식: `이름` 또는 `이름 전화번호뒷자리` (예: `김철수 1234`).
  - **자동 회원가입:** 명단에 없는 이름 입력 시 자동으로 신규 회원으로 등록.

#### C. 회원 관리 (Member Management)
- **회원 등록:** 이름과 전화번호 뒷자리(선택)로 간편 등록.
- **회원 등록:** 회원 출석 화면에서 검색 결과가 없을 경우, 해당 이름으로 즉시 신규 회원을 등록하고 출석 처리할 수 있습니다.
- **상세 이력:** 리포트 등에서 회원 이름 클릭 시 개인별 전체 출석 이력(날짜, 수업명, 시간)을 조회할 수 있습니다.

#### D. 리포트 및 통계 (Analytics)
- **월별 필터:** `YYYY-MM` 형식으로 데이터 필터링.
- **탭 기반 리포트:**
  - **대시보드:** 월간 총 출석 인원(UV) 및 최근 6개월 출석 추이 그래프를 제공하는 요약 탭.
  - **차수별 주간 출석:** 각 수업(차수)의 주차별 출석 횟수 통계.
  - **개인별 주간 출석:** 개인별 1~5주차 출석 현황 매트릭스.
  - **출석 순위:** 회원별 출석 횟수 랭킹.
  - **수업 참여 현황:** 월별 참여 인원이 많은 수업 순위.
  - **신규 회원:** 최근 6개월간 월별 신규 가입자 수 추이.
  - **장기 미출석:** 설정 기간 동안 활동이 없는 회원 목록.
### E. 보안 및 복구 (Security)
- **비밀번호 보호:** 관리자 모드 및 민감한 작업 시 비밀번호 확인.
- **마스터 복구 코드:** 
  - 비밀번호 분실 시를 대비한 고유 복구 코드(`RECOVERY-XXXX`) 제공.
  - 비밀번호 변경 메뉴 하단에서 확인 가능.
  - 분실 시 로그인 화면에서 코드를 입력하여 비밀번호를 `1234`로 초기화 가능.
- **데이터 백업 및 복구:** 
  - 관리자 도구에서 모든 데이터를 JSON 파일로 내보내(백업) 로컬에 저장할 수 있습니다.
  - **(권장) 백업 시 비밀번호를 설정하여 데이터를 암호화할 수 있습니다. 암호화된 파일은 복구 시 비밀번호가 필요하며, 분실 시 복구가 불가능합니다.**
  - 브라우저 데이터 삭제 등 만일의 사태에 대비해, 백업 파일을 다시 가져와(복구) 데이터를 원상복구할 수 있습니다. 복구 시 기존 데이터는 모두 삭제됩니다.

### F. 앱 설치 방법 (PWA)
이 앱은 PWA(Progressive Web App) 기술을 지원하여 네이티브 앱처럼 설치하고 사용할 수 있습니다.
- **PC:** Chrome, Edge 등 주소창 우측의 '앱 설치' 아이콘 클릭.
- **모바일 (Android/iOS):** 브라우저 메뉴에서 **'홈 화면에 추가'** 또는 **'앱 설치'** 선택.

### 3. 데이터 관리 (Data Persistence)
- **저장소:** **Google Firebase Firestore** (NoSQL Cloud DB).
- **저장소:** **IndexedDB** (브라우저 내장 NoSQL 데이터베이스).
- **구조:**
  - `members` 컬렉션: 회원 정보 문서.
  - `logs` 컬렉션: 출석 기록 문서.
- **초기화:** 관리자 헤더의 '초기화' 버튼을 통해 DB 내 모든 데이터를 영구 삭제할 수 있습니다.
- **초기화:** 관리자 '설정' 탭의 '초기화' 버튼을 통해 **Firebase 서버와 로컬 기기의 모든 데이터를 영구적으로 삭제**할 수 있습니다. 이 작업은 되돌릴 수 없으므로 신중해야 합니다.

## ⚠️ 주의 사항

1. **QR 코드:** 웹사이트 주소가 변경되면 QR 코드를 다시 인쇄해야 합니다.
2. **데이터 보존:** 모든 데이터는 브라우저에 저장됩니다. **같은 기기, 같은 브라우저**에서만 데이터가 유지됩니다. 다른 기기나 다른 브라우저에서는 데이터가 공유되지 않습니다.
3. **데이터 백업:** 중요한 데이터는 **정기적으로 '데이터 백업' 기능을 사용해 암호화하여 백업**하는 것을 강력히 권장합니다. **백업 시 설정한 비밀번호는 절대 잊지 마세요.**
